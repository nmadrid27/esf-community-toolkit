# Templates

Blank forms for each ESF practice. The **default** template for each artifact is the one described in the PRD. Institutional variants add YAML frontmatter for course workflows.

## Core Artifacts (PRD v1)

| Template | What it is | Notes |
|---|---|---|
| `position-statement.md` | **Default.** Three elements: stance, what matters most, what you will not compromise on. | PRD-canonical format. Use for any project. |
| `record-of-resistance.md` | **Default.** Three sections: What AI Suggested, Why I Rejected or Revised It, What I Did Instead. | PRD-canonical format. One file per decision. |
| `disclosure-statement.md` | Disclosure statement with short, standard, and detailed tiers. | Companion drafts this at milestones and project close; use the template for manual writing. |
| `five-questions-checklist.md` | Quick yes/no checklist for the Five Questions. | Run at review points from Phase 3 onward. |
| `session-log-template.md` | Per-session work log. | Companion generates these automatically on Claude Code. Use for manual tracking on conversation platforms. |

## Institutional Extensions

These add YAML frontmatter and metadata fields for course workflows. Same content structure as the defaults above.

| Template | Extends | When to use |
|---|---|---|
| `position-statement-template.md` | `position-statement.md` | Course requires frontmatter. Adds post-session drift check section. |
| `record-of-resistance-template.md` | `record-of-resistance.md` | Course requires frontmatter and numbered records. |

## AI Use Log

The PRD specifies the AI Use Log as auto-generated by the Companion from session data. These templates are for conversation-platform users who track manually.

| Template | When to use |
|---|---|
| `ai-use-log-lite-template.md` | First project. Lightweight session blocks with Five Questions checkboxes. |
| `ai-use-log.md` | Simple table format for tracking AI contributions across a project. |
| `ai-use-log-template.md` | Full version with verification table and pattern analysis. |
