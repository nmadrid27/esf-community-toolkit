# ESF Evolution Log

This file records how your ESF practice has diverged from the defaults, and why.
Each entry is a deliberate, reasoned adaptation — not an exception, but a documented part of how you work.

The Companion reads this file and adjusts its behavior to match your active entries.

---

## How to add an entry

The Companion will generate entries through the Framework Evolution Protocol.
You can also write entries directly using the format below.

<!-- Example entry — delete before use:

## Evolution Entry 2026-04-01

**Default behavior:** Position Statement written before any research or AI engagement
**My practice:** I do a brief survey pass (reading, not AI) first, then write the Position Statement
**Why:** My work is research-driven. Without a baseline of what's known, my initial position is too shallow to be a useful anchor. A one-hour survey pass produces a position I can actually defend.
**Consequences acknowledged:** My position reflects informed prior knowledge, not raw prior knowledge. This changes what the statement protects: it anchors against AI framing during the work, not against research defaults.
**Status:** active
