# Templates

Blank forms for each ESF practice. Pick the version that fits your situation.

## Position Statement

| Template | When to use |
|---|---|
| `position-statement.md` | Default. Clean and simple. Use for any project. |
| `position-statement-template.md` | When your course requires YAML frontmatter or you want the post-session drift check section. |

## Record of Resistance

| Template | When to use |
|---|---|
| `record-of-resistance.md` | Default. Simple three-section format: What AI Suggested, What I Did Instead, Why. |
| `record-of-resistance-template.md` | When your course requires YAML frontmatter and numbered records. Uses the same three-section structure with pre-filled metadata fields. |

## AI Use Log

| Template | When to use |
|---|---|
| `ai-use-log-lite-template.md` | First project. Lightweight: session blocks with Five Questions checkboxes. Start here. |
| `ai-use-log.md` | Standard. Simple table format for tracking AI contributions across a project. |
| `ai-use-log-template.md` | Full version with verification table, pattern analysis, and engagement level tracking. Use after your first project. |

## Other Templates

| Template | Purpose |
|---|---|
| `five-questions-checklist.md` | Quick yes/no checklist for the Five Questions. Run at review points. |
| `disclosure-statement.md` | Disclosure statement with short, standard, and detailed tiers. Attach to finished work. |
| `reflection-template.md` | End-of-project reflection covering position evolution, key decisions, and lessons learned. |
| `session-log-template.md` | Per-session work log. The Companion generates these automatically on Claude Code; use this template for manual tracking on other platforms. |
| `evolution-log-template.md` | Track how your ESF practice changes across multiple projects. Optional. |
