# Templates

Blank forms for each ESF practice. Use the version that matches your situation.

The **default** template for each artifact is the simple version — no frontmatter, works anywhere. Institutional variants add YAML frontmatter for course workflows.

---

## Position Statement

| File | Use when |
|------|----------|
| `position-statement.md` | **Default.** No frontmatter, no course fields. Works with any tool, including non-Claude-Code paths. |
| `position-statement-template.md` | Institutional. Includes frontmatter, course fields, and a post-session drift check. |

Both use the same three elements: My Stance, What Matters Most, What I Will Not Compromise On.

---

## Record of Resistance

| File | Use when |
|------|----------|
| `record-of-resistance.md` | **Default.** Simple format, no frontmatter. One file per decision. |
| `record-of-resistance-template.md` | Institutional. Frontmatter with numbered records. This is the format the ESF Companion uses when it creates Records of Resistance automatically. |

Both use the same section order: What AI Suggested → Why I Rejected or Revised It → What I Did Instead.

---

## AI Use Log

The Companion auto-generates AI Use Logs on Claude Code. These templates are for conversation-platform users who track manually. Three versions in order of complexity:

| File | Use when |
|------|----------|
| `ai-use-log.md` | Simplest. No frontmatter, no Five Questions. One table per session. Good for reconstruction on projects already in progress. |
| `ai-use-log-lite-template.md` | First projects. Frontmatter, Five Questions check per session, Records of Resistance table. |
| `ai-use-log-template.md` | Full version. Includes prompt evolution notes, pattern analysis, and percentage breakdown. |

## Other Templates

| File | Purpose | Status |
|------|---------|--------|
| `five-questions-checklist.md` | Standalone Five Questions audit. Run at major decision points, before submitting, or when something feels off. | Core |
| `disclosure-statement.md` | Disclosure with short, standard, and detailed forms. Companion drafts this on Claude Code; use the template for manual writing. | Core |
| `companion-state-template.md` | Runtime state file for the Companion. Tracks identity, active contexts, current project, and preferences. Installed by `/esf-onboarding`, not filled in manually. | Infrastructure |
| `project-brief-template.md` | Instructor-authored project brief. Includes ESF requirement fields (position-statement required, Records of Resistance minimum). For course contexts. | Instructor use |
| `project-plan.md` | User-authored work map: what you are making, why, milestones, open questions, and constraints from your Position Statement. Optional but useful for complex projects. | Optional |
| `session-log-template.md` | Per-session record of decisions, drift, and Records of Resistance. Auto-generated by the Companion at session end on Claude Code; use for manual tracking elsewhere. | Core |
| `reflection-template.md` | End-of-project reflection: position summary, kept/revised/rejected, Five Questions, disclosure. Phase 5 Reflect is Companion-guided on Claude Code; this template is for manual use. If you have already completed a full AI Use Log, the reflection adds end-of-project learning questions rather than re-documenting the same material. | Supplementary |
| `evolution-log-template.md` | Documents how your ESF practice has changed across projects. Not a project artifact: a practice artifact. | Not part of core ESF |
